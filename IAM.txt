🔐 EXPERIMENT 10: AWS IAM (A–Z COMPLETE GUIDE)
________________________________________
🧠 0. First understand the system (VERY IMPORTANT)
Before clicking anything:
IAM = Gatekeeper of AWS
It controls:
•	Who can log in
•	What they can access
•	What actions they can perform
Core entities:
Component	Meaning
User	Individual identity
Group	Collection of users
Policy	Permission rules
Role	Temporary access
________________________________________

🔧 Steps:

🔐 9. Concepts You MUST Understand (for viva)
________________________________________
🔹 IAM Policy
•	JSON document defining permissions
Example:
•	Allow S3 read
•	Deny EC2 delete
________________________________________
🔹 Least Privilege
Give ONLY required permissions, nothing extra
________________________________________
🔹 MFA (Multi-Factor Authentication)
•	Adds extra security layer (OTP, device)
________________________________________
🔹 Role vs User
User	Role
Permanent identity	Temporary access
Has credentials	No long-term credentials
________________________________________
